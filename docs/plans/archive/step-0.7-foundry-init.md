# Step 0.7 — Foundry Init for Contracts

> **Plan status:** Ready for execution.
> **Phase:** 0 — Project Setup.
> **ARCHITECTURE.md reference:** Phase 0, Step 0.7.

## Purpose

Three deliverables:

1. **`contracts/` directory initialized** via `forge init contracts/ --no-git`, then cleaned of boilerplate (Counter.* files and README.md deleted; `.gitkeep` added to empty directories).

2. **`contracts/foundry.toml` edited** — add `solc_version = "0.8.24"` and `evm_version = "cancun"` to the `[profile.default]` block generated by forge init.

3. **`contracts/.gitignore` created** — ignores `out/`, `cache/`, `broadcast/` (Foundry's build artifact and deployment record directories).

Additionally:

- **`forge-std` installed as a git submodule** at `contracts/lib/forge-std`, pinned to tag `v1.16.1`. This replaces the plain-cloned copy that `forge init --no-git` deposits. The `contracts/lib/forge-std/` clone from forge init is explicitly deleted before the submodule is added.
- **`.gitmodules`** is created automatically by `git submodule add`. It is committed as part of this step.

The `contracts/` directory does not exist yet in the Krax repo. This step creates it. The Phase 0 Gate item `cd contracts && forge build` succeeds is satisfied by this step.

## Files to create or modify

### Ordered execution sequence

**The order below is mandatory.** Running `git submodule add` before deleting the cloned `contracts/lib/forge-std/` produces a "destination path already exists" error. Deleting Counter.* before editing `foundry.toml` is arbitrary order-wise, but following the list top-to-bottom avoids any ambiguity.

1. Run `forge init contracts/ --no-git`
2. Delete `contracts/src/Counter.sol`, `contracts/test/Counter.t.sol`, `contracts/script/Counter.s.sol`, `contracts/README.md`
3. Add `.gitkeep` to `contracts/src/`, `contracts/test/`, `contracts/script/`
4. Delete `contracts/lib/forge-std/` (the plain-cloned copy from forge init)
5. Run `git submodule add --branch v1.16.1 https://github.com/foundry-rs/forge-std contracts/lib/forge-std`
6. Edit `contracts/foundry.toml`
7. Create `contracts/.gitignore`

### Step 5: Add forge-std as a git submodule

```bash
git submodule add --branch v1.16.1 https://github.com/foundry-rs/forge-std contracts/lib/forge-std
```

This command:
- Clones forge-std at tag `v1.16.1` (commit `620536fa5277db4e3fd46772d5cbc1ea0696fb43`) into `contracts/lib/forge-std/`
- Creates `.gitmodules` at the repo root with a submodule entry
- Stages both the submodule pointer and `.gitmodules` for commit

The submodule is tracked by a pointer in the outer git repo. `git submodule status contracts/lib/forge-std` should show the pinned commit SHA. New contributors must run `git submodule update --init` after cloning the Krax repo to populate `contracts/lib/forge-std/`.

`.gitmodules` is auto-created. Do not create it manually. The resulting `.gitmodules` will look like:

```ini
[submodule "contracts/lib/forge-std"]
	path = contracts/lib/forge-std
	url = https://github.com/foundry-rs/forge-std
	branch = v1.16.1
```

### Step 6 (edit existing): `contracts/foundry.toml`

`forge init` creates this file with the following content:

```toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]

# See more config options https://github.com/foundry-rs/foundry/blob/master/crates/config/README.md#all-options
```

Edit it to add `solc_version` and `evm_version`. The final content must be:

```toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]
solc_version = "0.8.24"
evm_version = "cancun"

# See more config options https://github.com/foundry-rs/foundry/blob/master/crates/config/README.md#all-options
```

**Edit, do not replace wholesale.** Insert the two new lines after `libs = ["lib"]` and before the comment line, preserving the comment. Use a targeted `str_replace`-style edit.

Rationale for each addition:
- `solc_version = "0.8.24"`: without this pin, `forge build` fetches the latest solc (0.8.34 in the scratch test). Pinning is mandatory for reproducible builds and matches the Solidity version declared in AGENTS.md ("Solidity 0.8.24+").
- `evm_version = "cancun"`: Ethereum mainnet is post-Dencun (Cancun). Krax targets Ethereum L1 for settlement. `cancun` is the correct EVM target as of May 2026.

Not added now (deferred to Phase 12 when real contracts exist):
- `optimizer = true` / `optimizer_runs` — optimizer tuning is meaningless without real code to profile.
- `via_ir = true` — IR-based compilation deferred until contracts have code that benefits from it.

## Open questions / coder follow-ups

**If `git submodule add` errors with "destination path already exists":** Step 4 (delete `contracts/lib/forge-std/`) was skipped or incomplete. Run `rm -rf contracts/lib/forge-std` and retry Step 5.

**If `git submodule add` errors with "already exists in the index":** A partially-staged submodule entry is present. Run `git rm --cached contracts/lib/forge-std` to unstage, then retry Step 5.

**If `forge build` downloads a solc version other than 0.8.24:** The `solc_version` key in `foundry.toml` is missing or misspelled. Verify with `grep solc_version contracts/foundry.toml` before investigating further.

**If `forge build` errors with "Source file not found" or similar after the Counter.* deletion:** forge-std is not properly initialized. Run `git submodule update --init contracts/lib/forge-std` and retry.

**If the scratch-test file tree does not match at runtime (forge init produces additional files):** Document the deviation, do not silently proceed. The plan is written against a verified scratch output; any new file from forge init needs an explicit keep/delete decision.
